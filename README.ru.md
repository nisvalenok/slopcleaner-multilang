# SlopCleaner Multilingual

[English](README.md) · [Deutsch](README.de.md) · **Русский**

Скилл убирает признаки ИИ-письма («слоп») из текста на **английском, немецком и русском** — и построен вокруг принципа, который большинство похожих инструментов не проговаривает.

## В чём беда обычного подхода

Хорошие англоязычные де-слоп-скиллы есть, есть и пара для азиатских языков. Почти у всех одна слепая зона: они считают признаки ИИ единым списком, который переводят с языка на язык. Так нельзя — **то, что в одном языке признак машины, в другом нормальная типографика**:

- **Тире** в английском (em-dash) — сильный признак. В немецком (Gedankenstrich) и русском это обычный знак с реальным весом: русское тире заменяет глагол-связку («Москва — столица»).
- **Длинные сложноподчинённые предложения** в английском читаются тяжело, а в немецком и русском это норма.
- **Причастные обороты, двоеточия, безличные конструкции** — каждый ложится в каждом языке по-своему.

Прогонишь английский чек-лист по всем трём — сотрёшь ровно то, что делает немецкую и русскую прозу живой.

## Что SlopCleaner делает иначе

Каждый язык разбирается на своих условиях. Общее **структурное ядро** (ложные деятели, пустые декларации, отсутствие позиции, вода во введении и заключении — признаки в любом языке) плюс **файл на каждый язык**, где перечислены две вещи:

1. признаки, типичные именно для этого языка, и
2. его **исключения** — конструкции, которые выглядят как признаки, но являются нормой языка, и трогать их нельзя.

Раздел исключений — то, что другие скиллы пропускают. Это разница между редактором, который знает язык, и скриптом «найти и заменить». (Конкретно: уберёшь каждое тире из немецкого текста — он станет жёстче, а не человечнее, потому что Gedankenstrich — нормальная немецкая типографика. Такие ошибки и кодируют исключения.)

## Честные границы

Это один из многих де-слоп-скиллов, а не категория из себя одного. Чем выделяется:

- **Немецкий и русский** в одном многоязычном скилле — пара языков, которой существующие инструменты не покрывают.
- **Исключения на каждый язык как заявленный принцип**, а не задним числом.
- **Честная маркировка источников.** Английские признаки опираются на корпусные исследования; немецкий и русский держатся в основном на наблюдениях практиков — и скилл это говорит (`[STUDY]` против `[PRACTITIONER]` по всему `sources.md`).
- **Не инструмент обхода детекторов.** Многие скиллы в этой нише продают «обойти GPTZero / Turnitin». Этот — нет. ИИ-детекторы на неанглийской прозе дают меньше ~40 %; эти списки служат суждению живого редактора, а не тому, чтобы заверить — или скрыть — происхождение текста.

Пишешь только на английском — узкий англоязычный скилл может подойти лучше. SlopCleaner оправдывает себя, когда работаешь сразу с этими языками и хочешь, чтобы каждый уважали отдельно.

## Структура

```
slopcleaner-multilang/
├── SKILL.md              # роутер: определить язык → ядро → файл языка
├── README.md
├── LICENSE
└── references/
    ├── core.md           # структурные признаки, не зависящие от языка
    ├── en.md             # английский (на корпусных данных)
    ├── de.md             # немецкий + исключения
    ├── ru.md             # русский + исключения
    └── sources.md        # источники + уверенность по каждому пункту
```

## Как это работает

1. Определить язык редактируемого текста (не язык разговора).
2. Применить структурное ядро.
3. Применить нужный файл языка, соблюдая его раздел ИСКЛЮЧЕНИЙ.
4. Прочитать вслух и судить на слух — чек-лист служит чтению, а не наоборот.

## Как добавить язык

Архитектура наращиваемая: новый язык — это новый файл плюс одна строка в роутере. Существующие языки не трогаются. Чтобы сделать это правильно:

1. **Сначала исследовать, не переводить.** Не прогоняй английский список через машинный перевод. Найди задокументированные наблюдения ИИ-признаков *в этом языке* (лингвистические работы, разборы редакторов, исследования детекторов). Сошлись на них.
2. **Создай `references/<lang>.md`** с двумя разделами:
   - **Признаки** этого языка (обороты, привычки пунктуации, структурные шаблоны).
   - **ИСКЛЮЧЕНИЯ** — конструкции, которые выглядят как признаки, но являются нормой языка. Раздел обязателен и важнее всего. Без него файл будет портить хороший текст.
3. **Добавь одну строку в `SKILL.md`**, направляющую язык к его файлу.
4. **Допиши источники в `references/sources.md`**, каждый с меткой `[STUDY]`, `[COMPENDIUM]` или `[PRACTITIONER]` по силе доказательства.
5. **Не переноси правила между языками.** Если правило (например, «убрать тире») только английское — оно остаётся в `en.md`. В ядре только то, что действительно не зависит от языка.

Язык с малой исследовательской базой всё равно может получить файл ИСКЛЮЧЕНИЙ и опереться на ядро; это лучше переведённого чек-листа, который язык уплощает.

## Установка

Положи папку `slopcleaner-multilang/` в каталог скиллов (папка скиллов Claude или туда, откуда твой агент их грузит). Файлы-справочники подгружаются по необходимости.

## Участие

Issues и pull request приветствуются — особенно новые файлы языков по правилу «сначала исследовать, не переводить» и лучше обоснованные признаки для немецкого и русского (две самые тонкие базы). Раздел ИСКЛЮЧЕНИЙ у каждого языка оставляй нетронутым — это сердцевина замысла.
